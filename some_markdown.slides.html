Hashtables are basically an array that links a key to a specific data value(s), where each key is unique. They are data structure that efficiently implements search/lookup operations. It aims to divide the whole search space into well-defined regions, so we dont have to search linearly through the entire collection to find an element.

Hashtable Techniques -

  • Hashing
  • Hash function
    Key values are converted to integer values using hashing technique. And we use the hash function to find the required key-value pair. So, we reduce the search time from linearly to directly searching in the bucket.

Ideally for perfect hashing we have only 1 value for each array index. So that we can directly find the index after hashing and get a time complexity of O(1).

This is practically impossible.

Indexing After a page is crawled, Google tries to understand what the page is about. This stage is called indexing and it includes processing and analyzing the textual content and key content tags and attributes, such as elements and alt attributes, images, videos, and more.</p> <p>During the indexing process, Google determines if a page is a duplicate of another page on the internet or canonical. The canonical is the page that may be shown in search results. To select the canonical, we first cluster the pages that we found on the internet that have similar content, and then we select the one that's most representative of the group. The other pages in the group are alternate versions that may be served in different contexts, like if the user is searching from a mobile device or they're looking for a very specific page from that cluster.</p> <p>Google also collects signals about the canonical page and its contents, which may be used in the next stage, where we serve the page in search results. Some signals include the language of the page, the country the content is local to, the usability of the page, and so on.</p> <p>The collected information about the canonical page and its cluster may be stored in the Google index, a large database hosted on thousands of computers. Indexing isn't guaranteed; not every page that Google processes will be indexed.</p> <p>Indexing also depends on the content of the page and its metadata.
